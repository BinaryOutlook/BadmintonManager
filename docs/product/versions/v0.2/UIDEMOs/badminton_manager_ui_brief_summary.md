# Badminton Manager V0.2 UI Brief

## Product Framing
Badminton Manager is a coaching and tournament management simulation. The UI must feel like a coaching console, a tournament command center, or a match analysis desk.

## Design Goals
*   **Vibe:** Modern, solid, premium, data-dense, sport-specific, serious.
*   **Inspiration:** Football Manager-style dashboard (but not a literal imitation).
*   **Mood:** Dark grounded surfaces, strong visual hierarchy, compact readability.

## Visual Direction
*   **Surfaces:** Layered dark surfaces (charcoal, graphite, slate, deep blue-black). No glassmorphism.
*   **Accents:** Restrained green or cyan for functional states.
*   **Typography:** Editorial, sharp, deliberate. Clear hierarchy for dense data.
*   **Shapes:** Structured, substantial panels with visible borders and subtle rounding.

## Required Screens
1.  **Global App Shell:** Persistent identity and player context.
2.  **Setup Screen:** Athlete selection and tactical briefing.
3.  **Tournament Overview:** Bracket desk and scouting board.
4.  **Live Match Screen:** The "Command Center" with scoreboard, commentary, and team talks.
5.  **Run Complete Screen:** Tournament recap and reset point.

## UX Principles
*   **Readability:** Immediate understanding of tournament state and active tactics.
*   **Match Drama:** Tension between sets and pressure in the commentary feed.
*   **Product Identity:** Focus on coaching and management, not action gameplay.
