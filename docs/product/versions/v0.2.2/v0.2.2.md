# V0.2.2 Roster Draw Expansion

Status: Active
Owner: BinaryOutlook
Last updated: 2026-05-05
Iteration type: Content expansion and tournament variety system

## 1. Summary

`v0.2.2` expands the local athlete pool and changes tournament creation so every run can feel
less predictable without changing the 16-player knockout structure.

The headline change is:

```text
32-athlete pool -> draw 16 event entrants -> reseed field -> Round of 16 bracket
```

The manager still plays a compact tournament. The novelty comes from who enters the field.

## 2. Release Thesis

The tournament loop should stay readable:

$$
\text{select athlete} \rightarrow \text{scout opponent} \rightarrow \text{choose tactic}
\rightarrow \text{manage match} \rightarrow \text{review run}
$$

But the roster should not feel like the same closed room every run.

`v0.2.2` adds a larger roster and keeps the bracket fixed at 16 entrants, so the game gains
variety without becoming a larger career-mode system.

## 3. Delivered Scope

### 3.1 Expanded athlete pool

The local content pool now contains 32 fictional athletes in `src/game/content/players.ts`.

The added group includes six **Trophy Titans** with sharper identity profiles and title-style
names:

- Grand-Slam Southpaw: final-boss hybrid, target OVR 92
- Nordic Tower: structured rear-court power, target OVR 91
- Greatwall Dragon: defensive counter-fortress, target OVR 90
- Three-Lung Dynamo: endless rally engine, target OVR 90
- Eight-Crown Monarch: old-school rhythm controller
- Backhand Mirage: deceptive wrist and backhand shot-maker

Only Trophy Titans should use this distinctive naming convention. Their names are deliberately
mythic because they represent legacy playstyle archetypes.

All other added athletes use plausible fictional real-name constructions. Their playstyle identity
belongs in `styleLabel`, `traits`, scouting, and ratings rather than in the athlete name itself.

### 3.2 Sixteen-player event draw

Tournament creation now draws exactly 16 entrants from the larger player pool.

The draw rule is:

- always include the managed athlete
- randomly draw the remaining 15 entrants from the local roster pool
- use the run seed so the draw remains deterministic
- reseed the selected field by content seed order
- apply the existing 16-player bracket order

This keeps the tournament model compact:

```text
larger pool
    |
    v
managed athlete + 15 drawn entrants
    |
    v
fixed R16 -> QF -> SF -> F bracket
```

### 3.3 Consistent roster OVR display

The setup roster cards now use the same four-metric OVR formula as the selected athlete dossier:

$$
\text{OVR} = \operatorname{round}\left(
\frac{\text{power} + \text{speed} + \text{stamina} + \text{control}}{4}
\right)
$$

That keeps player selection, matchup comparison, and tests aligned.

## 4. Product Rules

`v0.2.2` does not change the player fantasy.

The player is still the coach, not the athlete. Roster novelty should support scouting and tactical
planning, not turn the game into direct control or licensed athlete simulation.

The expanded draw system must preserve:

- deterministic outputs for identical seeds and choices
- a 16-player tournament field
- local-first content and persistence
- fictional player identity
- title-style names reserved for Trophy Titans
- simulation logic outside React components

## 5. Balance Notes

The expanded roster intentionally includes stronger top-end players.

The first tuning pass should watch:

- whether the four highest-rated additions suppress upset frequency too much
- whether defensive legends make matches drag in a fun way or a slow way
- whether the selected athlete's opening draw feels varied across repeated runs
- whether scouting text has enough flavor to distinguish the new playstyles

## 6. Verification Expectations

Keep these commands green:

- `npm run typecheck`
- `npm run test`
- `npm run build`
- `npm run test:e2e`

Roster-specific checks should cover:

- the local pool contains 32 athletes
- only the six Trophy Titans use title-style naming
- the tournament field contains exactly 16 unique entrants
- the managed athlete is always included in the drawn event
- the same run seed produces the same event field
- different run seeds can produce different event fields
- the headline legend OVR targets remain stable

## 7. Relationship To V0.2.1

`v0.2.1` remains the command-center patch for interaction fixes and overview rearrangement.

`v0.2.2` is the feature-bearing roster expansion release. Its job is to make the same compact
tournament loop feel fresher through content depth and deterministic draw variety.
