# V0.2.1 Patch Release

Status: Active
Owner: BinaryOutlook
Last updated: 2026-05-05
Iteration type: Bug-fix, UI rearrangement, and pre-feature cleanup

## 1. Summary

`v0.2.1` is a stabilizing patch on top of the `v0.2` command-center build.

The goal is not to add a large new system. The goal is to make the current loop easier to read,
easier to operate, and safer to extend before the next feature update.

This version fixes a few interaction and rendering bugs, rearranges the command-center UI around
clearer match preparation, and creates breathing room for small, innovative, and redefining
features in the next release.

## 2. Patch Thesis

The command center should now feel less like a set of stacked panels and more like a coach's
pre-match desk.

The player should be able to:

- compare their athlete against the next opponent at a glance
- lock a tactic without losing sight of the matchup
- read the full knockout path without the tree feeling visually broken
- trust that interval choices and shell controls respond when clicked

## 3. Delivered Fixes

### 3.1 Between-set team talk

Fixed the bug where team-talk cards could be clicked but appeared unselectable.

The root cause was that the engine queued `pendingTalkA` or `pendingTalkB`, while the UI only
displayed active directive state. `v0.2.1` now:

- allows talks only during intermission
- shows the selected talk as active
- confirms the queued talk in the live feed
- applies the queued talk when the next set opens

### 3.2 Clickable shell controls

The top navigation and left-side command menu now behave as controls instead of static labels.

The sidebar can surface compact command options such as:

- live desk entry
- tactic switching
- athlete status
- event status
- tactical intel and reset actions

### 3.3 Full knockout tree rendering

The knockout tree now renders as a true binary bracket:

```text
Round of 16 -> Quarter-Finals -> Semi-Finals -> Final
8 matches   -> 4 matches      -> 2 matches   -> 1 match
```

The layout uses a shared row grid so child matches align into their parent match slots. Future
rounds remain visible as placeholders until the tournament model creates concrete matches.

### 3.4 Tournament command-center rearrangement

The pre-match screen now uses a denser, more useful composition:

- next opponent is a head-to-head comparison
- managed athlete appears on the left
- opponent appears on the right
- overall, power, speed, stamina, and control are compared in a compact center lane
- tactic lock-in sits on the right side of the matchup instead of underneath it
- the knockout tree starts closer to the matchup and tactic panels

## 4. Design Direction

`v0.2.1` keeps the dark command-center identity, but reduces wasted motion in the overview screen.

The practical layout rule is:

$$
\text{matchup read} + \text{tactic lock} \rightarrow \text{bracket context}
$$

That order should guide near-term overview changes unless a later release introduces a stronger
pre-match workflow.

## 5. Preparation For The Next Update

This patch intentionally rearranges the UI before deeper feature work. The likely next target is
`v0.2.2`.

The next update should favor small ideas with high identity impact rather than a broad system
expansion. Good candidates are:

- richer tactical intel explanations tied to real engine state
- sharper directive variety with clearer trade-offs
- lightweight opponent pattern reads
- post-match insights that explain one decisive turning point
- compact feature moments that make the coaching fantasy feel more original

The next release should avoid becoming a large career-mode push too early. The better path is to
add a few redefining surfaces that make the existing tournament loop feel smarter.

## 6. Verification Expectations

Before calling `v0.2.1` stable, keep these commands green:

- `npm run typecheck`
- `npm run test`
- `npm run build`
- `npm run test:e2e`

Browser QA should include:

- starting a tournament from setup
- reading the head-to-head comparison
- selecting a tactic from the right-side lock-in
- verifying the full knockout tree is aligned through the final
- reaching a set break and selecting a team talk

## 7. Relationship To V0.2

`v0.2` remains the major command-center release packet.

`v0.2.1` is the active patch release that documents the bug fixes, rearranged overview UI, and
stabilization work needed before the next feature-bearing version.
